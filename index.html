Semantic meaning

All elements in HTML (except for div and span ) have a semantic meaning. This semantic meaning is interpreted by browsers, screen readers, and SEO crawlers.

Browsers use this semantic value to properly display the elements and to bind interactions.

Screen readers also use this semantic value to bind additional different keyboard shortcuts and interactions.

Lastly, SEO crawlers use this meaning to measure importance of content and define relationships.

Worst Practice

Don’t use a div, span, or some other non-interactive element. If you cannot tab to it with a keyboard, it’s probably a terrible idea to use. Adding role="button" doesn’t help, that makes it worse.

What about our projects?

Buttons

  • Receive keyboard focus by default
  • “Click” with the Space key
  • Submit form data to a server / Reset a form
  • Instruct a screen reader with the implicit button role
  • Opening a modal window
  • Playing media content
  • Triggering a popup menu
  • Toggling an interface

Links

  • Create hypertext, a network of online resources
  • Navigate the user to a new page or view
  • Change the URL
  • Cause a browser redraw/refresh
  • Support page jumps with internal href attributes
  • Allow opening in new windows (and back in the day, framesets)
